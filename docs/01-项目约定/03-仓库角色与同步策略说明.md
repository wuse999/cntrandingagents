# 仓库角色与同步策略说明

## 1. 文档用途

- 固定 `raw / cntrandingagents / GitHub / Gitee / Ubuntu / Windows` 各自的角色。
- 固定代码、文档、tag、release 的同步路径。
- 防止把 `raw` 和 `cntrandingagents` 混成一条线。

## 2. 当前仓库与环境角色

### 2.1 `raw`

- 地址基线：官方 `TauricResearch/TradingAgents`
- 本地路径：`D:\tranding\tranding_agents_cn\raw`
- 角色：上游镜像仓
- 用途：拉取官方更新、查看官方基线、对照汉化差异

### 2.2 `cntrandingagents`

- Windows 本地路径：`D:\tranding\tranding_agents_cn\cntrandingagents`
- Ubuntu 主工作树：`~/projects/cntrandingagents`
- 角色：汉化主线仓
- 用途：维护文档汉化、CLI 对外文案汉化和发布资产

### 2.3 `upstream`

- 地址：`git@github.com:TauricResearch/TradingAgents.git`
- 角色：官方同步源
- 用途：检查官方更新、拉取官方变化

### 2.4 `origin`

- 规划地址：`git@github.com:wuse999/cntrandingagents.git`
- 角色：主仓与主发布仓
- 用途：主分支、主 tag、主 Release、主版本历史

### 2.5 `gitee`

- 规划地址：`git@gitee.com:frpnat/cntrandingagents.git`
- 角色：镜像仓与国内发布源
- 用途：国内访问、国内镜像下载、镜像 release

### 2.6 Ubuntu 开发机

- 登录：`ssh wuse2060@192.168.3.135`
- 主工作树：`~/projects/cntrandingagents`
- 角色：唯一正式 Git / 构建 / 发布执行面

### 2.7 Windows 本地镜像

- 路径：`D:\tranding\tranding_agents_cn\cntrandingagents`
- 角色：镜像、副本、脱机参考

## 3. 当前同步总原则

- 官方事实源：`upstream`
- 汉化事实源：Ubuntu 主工作树中的 `cntrandingagents`
- 主仓事实源：GitHub `origin`
- 国内镜像源：Gitee `gitee`
- Windows 不再承担主事实源职责

## 4. 默认同步路径

### 4.1 官方同步

- 先从 `upstream` 获取官方更新。
- 先对照 `raw` 与当前汉化差异。
- 再把需要的同步内容引入 `cntrandingagents`。

### 4.2 汉化主线同步

- 日常改动发生在 Ubuntu 主工作树 `~/projects/cntrandingagents`。
- 正式同步先推 GitHub `origin`。
- GitHub 正常后再推 Gitee `gitee`。

### 4.3 Windows 同步

- Windows 不承担日常同步主线。
- Windows 本地目录只作为镜像和应急副本。

## 5. 上游同步规则

- 任何官方更新先看是否影响文档、CLI 文案或对外展示文本。
- 能直接吸收的变更优先吸收。
- 会改变行为的变更在本仓只记录，不自行做功能增强。
- 每轮上游同步都应留下记录，写入 `docs/03-上游同步`。

## 6. 发布同步规则

1. 在 Ubuntu 主工作树完成汉化验证。
2. 提交并推送 GitHub `origin`。
3. 创建 GitHub tag 和 GitHub Release。
4. 同步 Gitee 分支、tag 和 Gitee Release。
5. 确认 GitHub / Gitee 下载入口都可用。

## 7. 关于 `raw` 与 `cntrandingagents` 的边界

- 不在 `raw` 中推进汉化主线。
- 不在 `raw` 中积累你自己的长期修改。
- `raw` 只保留上游镜像职责。
- 真正的汉化改动只进 `cntrandingagents`。

## 8. 明确禁止的做法

- 不把 `raw` 当正式汉化仓。
- 不把 Gitee 当与 GitHub 并行的双主仓。
- 不让 Windows 和 Ubuntu 并行维护两套主线。
- 不用“直接在官方仓上改”的方式长期维护汉化版。

## 9. 一句话结论

`cntrandingagents` 当前采用“官方上游单独镜像、汉化主线单独维护、GitHub 主仓、Gitee 镜像”的分层同步策略。
