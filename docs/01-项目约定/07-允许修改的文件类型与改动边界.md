# 允许修改的文件类型与改动边界

## 1. 文档用途

- 固定 `cntrandingagents` 当前允许修改的范围。
- 防止汉化过程中混入行为改造和功能扩展。

## 2. 当前允许修改的文件类型

- `README.md`
- 安装文档、使用文档、项目说明文档
- CLI 帮助文本和 CLI 对外提示文案
- 用户可见的错误提示、状态提示、菜单标题和步骤说明
- 示例文档、示例说明、截图说明
- 项目内用于展示和说明的对外文案

## 3. 当前优先汉化的内容

- 仓库首页 README
- 安装与运行说明
- CLI 交互文案
- 模型与配置说明
- 项目角色和架构说明
- 帮助文本与错误提示

## 4. 谨慎修改的内容

- 注释
- 默认配置说明
- 示例代码中的说明文字
- 输出格式标题

这些内容可以改，但前提是不改变程序行为和接口语义。

## 5. 默认禁止修改的内容

- Agent 图编排逻辑
- 数据流实现
- LLM 客户端逻辑
- 默认模型选择逻辑
- 反射、信号处理、debate、risk 管理等行为链路
- 数据源、工具、接口、缓存、数据库等任何功能扩展

## 6. 什么叫“只改文案不改行为”

符合以下条件才算“只改文案”：

- 参数名不变
- 返回值结构不变
- 控制流不变
- 工具调用路径不变
- 模型选择逻辑不变
- 程序运行结果语义不变

## 7. 需要单独评审的改动类型

- 任何涉及 prompt 内容大改的修改
- 任何涉及结构化输出变化的修改
- 任何涉及默认配置行为变化的修改
- 任何不是纯文案替换的代码改动

## 8. 超边界改动的处理方式

- 先停止提交到 `cntrandingagents`
- 把需求标记为“功能增强”
- 新建项目和新仓库继续推进

## 9. 一句话结论

只要一项改动开始改变运行行为，它就已经超出了 `cntrandingagents` 的汉化边界。
