# 发布与镜像规则

## 1. 文档用途

- 固定 `cntrandingagents` 的 GitHub / Gitee 发布口径。
- 说明哪个仓库是主发布源，哪个仓库承担国内镜像发布。

## 2. 当前发布原则

- GitHub 是主发布仓。
- Gitee 是国内镜像发布仓。
- GitHub 先发，Gitee 后跟。
- tag 与版本号必须保持一致。

## 3. GitHub 主发布规则

- 主分支、主 tag、主 Release 都以 GitHub 为准。
- 发布说明以 GitHub Release 页面为主口径。
- 若两侧信息不一致，以 GitHub 为主。

## 4. Gitee 镜像发布规则

- 先同步代码，再同步 tag，再同步 release。
- Gitee 主要承担国内访问与国内下载场景。
- Gitee 页面与附件应尽量与 GitHub 保持一致。

## 5. 版本号与 tag 约定

- 推荐版本格式：`vX.Y.Z-cntrandingagents.N`
- 所有 tag 在 GitHub 与 Gitee 两边保持同名
- 不允许 GitHub 与 Gitee 出现同版不同 tag 名

## 6. Release 内容边界

- 只发布汉化版本说明
- 不宣称新增官方不存在的功能
- 不把“汉化”包装成“重构版”或“增强版”
- 说明应明确本仓定位是汉化仓

## 7. 需要同步到 Gitee 的内容

- 主分支代码
- 对应 tag
- 对应 release 标题与说明
- 对应 release 附件

## 8. 发布后核对清单

1. GitHub `main` 已更新
2. GitHub tag 已可见
3. GitHub Release 已可访问
4. Gitee `main` 已更新
5. Gitee tag 已可见
6. Gitee Release 已可访问
7. GitHub 与 Gitee 的版本号一致

## 9. 下载入口口径

- GitHub 为主下载源
- Gitee 为国内备用下载源
- 如后续增加下载页，需同时说明两侧可用性

## 10. 一句话结论

`cntrandingagents` 的发布链路必须始终保持“GitHub 主发布、Gitee 镜像发布”的单主线口径。
