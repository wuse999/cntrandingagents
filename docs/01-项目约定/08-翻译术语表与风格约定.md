# 翻译术语表与风格约定

## 1. 文档用途

- 统一 `cntrandingagents` 项目的中英文术语。
- 固定文档、README、CLI 帮助和对外展示文本的翻译风格。

## 2. 翻译总体原则

- 优先忠实，不为“看起来更本地化”而改写行为含义。
- 优先统一，同一术语在全仓尽量只保留一种主译法。
- 对用户可见文案可以更自然，但不能失真。
- 对配置项、参数名、模型名、库名默认保留原文。

## 3. 项目名与产品名写法

- 项目正式名：`TradingAgents`
- 汉化项目名：`cntrandingagents`
- 文档中推荐写法：`TradingAgents 中文汉化版` 或 `TradingAgents 汉化仓`

## 6. CLI 与交互文案风格

- 用词直接、简洁、可操作。
- 尽量使用“请选择 / 输入 / 当前 / 已完成 / 失败 / 继续”等自然表达。
- 不故意堆砌金融黑话。
- 错误提示优先告诉用户下一步怎么做。

## 7. README 与文档风格

- 先说明项目是什么，再说明如何安装和如何使用。
- 中文句子优先自然流畅，不机械逐字翻译。
- 关键命令、路径、环境变量保留英文原样。
- 需要保留官方术语时，可首次出现时中英并列。

## 8. 中英混排规则

- 配置项、环境变量、命令、路径使用原文。
- 模型名称、库名、类名、函数名保留原文。
- 普通说明文字使用中文。

## 9. 暂不翻译的内容

- 函数名
- 类名
- 包名
- 模型 ID
- 第三方库名
- 参数名与配置键名

## 10. 术语更新规则

- 若发现同一术语已有多个译法，优先回收为一套统一说法。
- 新术语先补进本表，再进入正文。
- 如官方术语发生变化，应先更新本表，再批量修正全文。

## 11. 一句话结论

`cntrandingagents` 的翻译目标不是“看起来像全新产品”，而是“让中文用户清楚理解原项目，同时尽量不偏离官方语义”。
