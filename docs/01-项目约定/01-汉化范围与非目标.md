# 汉化范围与非目标

## 1. 文档用途

- 固定 `cntrandingagents` 的唯一目标。
- 明确哪些改动属于“汉化”，哪些改动已经超出本仓职责。
- 防止后续把本仓逐渐演变成“功能增强版 TradingAgents”。

## 2. 项目目标

`cntrandingagents` 的目标是基于官方 `TradingAgents` 做长期可维护的中文汉化版本，帮助中文用户更容易理解、安装、使用和演示原项目，同时尽量保持与官方主线直接同步的能力。

## 3. 本仓只做什么

- README、安装说明、使用说明的中文化。
- CLI 帮助文本、交互提示、错误提示等对外展示文案的中文化。
- 文档目录、示例说明、截图说明等资料的中文化。
- 术语统一、翻译风格统一、中文展示口径统一。
- 必要时补充少量不改变行为的注释、标题、说明文字。

## 4. 本仓明确不做什么

- 不新增 A 股、港股、美股专用功能。
- 不新增 Web 平台、API、后台管理能力。
- 不改数据流逻辑、图结构逻辑、Agent 行为逻辑。
- 不做“顺手优化”式功能重构。
- 不把内部推理系统、策略逻辑和功能边界改成中文版特有实现。

## 5. 哪些改动算“汉化”

- 把英文说明文字译成中文。
- 把 CLI 中用户能看到的英文文案换成中文。
- 把示例中的说明部分换成中文。
- 把术语改成统一中文表达，但不改变参数和程序行为。
- 在不改变执行结果的前提下，补充中文标题、章节说明和帮助信息。

## 6. 哪些改动算“功能增强”

- 修改默认模型选择逻辑。
- 新增或重写数据源。
- 修改图中节点顺序、节点职责或 debate 逻辑。
- 把 Agent 内部 prompt 改成完全不同的一套中文策略。
- 新增 A 股 ticker 规则、A 股交易日历、A 股数据适配。
- 加入 Web UI、数据库、缓存、报表系统等新能力。

## 7. 发现超出范围时如何处理

- 若需求只和汉化有关，继续在 `cntrandingagents` 中处理。
- 若需求已经涉及行为变化，停止在本仓推进。
- 若需求明确属于新功能，单独新建项目和新仓库。
- 不允许在本仓先混进去，再事后解释为“只是顺手做了点优化”。

## 8. 当前阶段优先级

当前阶段优先级固定为：

1. 文档汉化
2. CLI 对外文案汉化
3. 帮助文本与展示文本汉化
4. 术语表与风格统一
5. 上游同步机制固化

## 9. 一句话结论

`cntrandingagents` 是“汉化专用仓”，不是“中文版功能增强仓”；只要改动开始影响行为，它就不该继续留在这个仓库里。
