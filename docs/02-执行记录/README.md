# 执行记录入口

## 1. 文档用途

- 统一记录 `cntrandingagents` 的真实执行动作。
- 记录每天做了哪些汉化、改了哪些文件、验证了什么、遇到了哪些问题。

## 2. 这里记录什么

- 某一轮汉化的目标
- 某一轮汉化修改的文件
- 某一轮验证结果
- 某一轮推送或发布前后的结论
- 当前阻塞和下一步待办
- 长期重复出现、值得回收为固定经验的踩坑与处理方式

## 3. 这里不记录什么

- 已经完全固化的长期规则
- 协作边界
- 术语表
- 仓库角色定义

这些内容应写回 `docs/01-项目约定`。

## 4. 推荐命名方式

- `YYYY-MM-DD-事项名-执行记录.md`

## 5. 记录要求

- 写明目标
- 写明改动范围
- 写明验证结果
- 写明问题和后续动作
- 如出现重复性问题，补入长期踩坑台账
- 如形成长期规则，回写 `docs/01-项目约定`

## 7. 一句话结论

执行记录只写真实发生过的动作，不写抽象规则。
