## 6. 新增踩坑与处理

- 本轮新坑 1：Ubuntu 正式主工作树目录在当前环境中并不存在，无法直接进入既定路径执行正式 Git 收口。
  - 临时处理：先在 WSL 中创建 `/home/wuse888/projects/cntrandingagents`，再从 GitHub 正式仓克隆。
- 本轮新坑 2：WSL 当前环境没有可直接用于 GitHub / Gitee 的私钥，原生 `ssh` 会报 `Permission denied (publickey)`。
  - 临时处理：在 WSL Git 操作中显式使用 `GIT_SSH_COMMAND='/mnt/c/Windows/System32/OpenSSH/ssh.exe'`，借用 Windows 侧已可用的 SSH 能力完成远端访问。
- 本轮新坑 3：若整仓从 Windows `rsync` 到 Ubuntu，会把 Windows 工作副本中的行尾差异一并带入，导致正式主工作树看起来像“全仓被改动”。
  - 临时处理：先把 Ubuntu 仓库重置为干净状态，再仅同步本轮实际改动文件清单。
- 本轮新坑 4：PowerShell 调用 `wsl.exe` 时，终端会持续夹带 NAT/localhost 提示与乱码噪音，容易让人误判中文文件损坏。
  - 临时处理：不把终端输出直接当作文件状态依据；Windows 侧读取中文文件时强制使用 `Get-Content -Encoding UTF8`，并以 `compileall`、Git diff 与 IDE 实际文件内容交叉确认。
