# 长期踩坑与经验回收

## 1. 文档用途

- 记录 `cntrandingagents` 在多次执行中反复出现、值得长期保留的踩坑与经验。
- 为后续汉化、同步、发布和跨机器协作提供“少走回头路”的固定台账。
- 把“临时解决过一次”的问题，逐步升级成“以后默认不会再踩”的规则。

## 2. 填写规则

- 这里只记录可重复发生、值得长期保留的问题和经验。
- 一次性动作仍写入当天的执行记录。
- 同类问题再次出现时，不要重开一份新文档，优先回到本台账补充状态与处理。
- 若某条经验已升级为长期规则，必须在“回写位置”中注明对应文档，并把状态标记为“已固化”。

## 3. 状态说明

- `观察中`：已经识别，但还未完全固化到长期规则。
- `已固化`：已经回写到治理文档、模板或固定流程。
- `已淘汰`：曾经有效，但当前规则或环境已变化，不再作为默认经验。

## 5. 经验升级规则

满足以下任一条件时，必须把条目升级到长期规则或模板：

- 同类问题再次出现。
- 问题涉及 Windows 与 Ubuntu 协作。
- 问题涉及 Git、远端、权限、凭据、编码或发布。
- 问题已经有稳定、可复制的处理办法。

## 6. 使用方式

- 每轮工作结束前，回看当天执行记录，判断是否有内容应补入本台账。
- 若本台账新增条目，执行记录中也要写明“已回收至长期台账”。
- 若本台账条目已固化到治理文档，对应执行记录中应写明回写文档路径。

## 7. 一句话结论

执行记录负责写“这次发生了什么”，本台账负责写“以后别再怎么踩”，两者缺一不可。
