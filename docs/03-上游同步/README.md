# 上游同步入口

## 1. 文档用途

- 统一记录每轮官方 `TradingAgents` 同步的实际情况。
- 帮助后续回看“某次同步为什么这么处理”。

## 2. 这里记录什么

- 上游基线版本
- 本地汉化基线
- 同步范围
- 冲突文件
- 冲突处理方式
- 对汉化差异的影响
- 同步后验证结果

## 3. 什么时候新增一条记录

- 每次正式同步官方更新时
- 每次处理上游冲突时
- 每次做“吸收官方变化 + 补齐汉化”时

## 4. 推荐命名方式

- `YYYY-MM-DD-upstream-sync.md`

## 5. 模板文件

- [模板-upstream-sync.md](d:/tranding/tranding_agents_cn/cntrandingagents/docs/03-上游同步/模板-upstream-sync.md)

## 6. 一句话结论

上游同步记录的目标不是写流水账，而是保留“为什么这样同步”的决策依据。
